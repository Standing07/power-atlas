# 🚀 部署指南（手把手教學，不用會寫程式）

這份教學帶你把「全球電力地圖」放到網路上，讓全世界都能瀏覽。你已經有 **Zeabur** 帳號與伺服器訂閱，所以流程是：**把程式碼放上 GitHub → 在 Zeabur 匯入部署**。之後每次程式碼或資料更新，網站都會自動重新部署。

---

## 第一步：申請 GitHub 帳號並上傳程式碼

GitHub 是全世界最大的程式碼儲存平台，也是「每月自動更新資料」機制的基地（Zeabur 會直接從 GitHub 抓最新版本部署）。

1. 到 https://github.com/signup 申請帳號（免費；已有帳號直接登入）。
2. 登入後，點右上角「＋」→「New repository」。
   - Repository name 輸入：`power-atlas`
   - 選 **Public**（公開）或 **Private**（私人）都可以，Zeabur 兩種都能部署
   - 其他都不要勾，按「Create repository」。
3. 打開電腦的「終端機」（Terminal）App，貼上以下指令（把 `你的帳號` 換成你的 GitHub 使用者名稱）：

```bash
cd "/Users/sting/AI Projects/power-atlas"
git remote add origin https://github.com/你的帳號/power-atlas.git
git push -u origin main
```

4. 第一次推送會要求登入 GitHub——依照畫面指示用瀏覽器授權即可。
5. 重新整理 GitHub 頁面，看到一堆檔案就成功了 ✅

> 💡 如果 `git push` 出現權限錯誤，最簡單的解法是安裝 GitHub 官方工具：`brew install gh`，然後執行 `gh auth login` 依指示登入，再重新 `git push -u origin main`。或者直接請 Claude 幫忙。

---

## 第二步：在 Zeabur 部署上線

1. 到 https://dash.zeabur.com 登入你的 Zeabur 帳號。
2. 點「**建立專案**」（Create Project）——區域可選你已訂閱的伺服器（Osaka），或 Zeabur 提供的免費/共用區域。
   > 💡 這個網站是純靜態網頁，資源需求極低，放在你現有的伺服器上綽綽有餘，不需要另外加購。
3. 進入專案後，點「**新增服務**」（Add Service）→「**Git**」。
4. 第一次會請你授權 Zeabur 存取 GitHub——按「Authorize」，然後選擇 `power-atlas` 這個 repository。
5. Zeabur 會自動偵測這是 Vite 靜態網站（建置指令 `npm run build`、輸出資料夾 `dist`），直接按部署即可，不用改任何設定。
6. 等 1–2 分鐘部署完成，到服務的「**網路**」（Networking）分頁 →「**產生網域**」（Generate Domain），輸入一個名稱（例如 `power-atlas`），就會得到 `https://power-atlas.zeabur.app` 這樣的公開網址 🎉
7. （選配）想用自己的網域：同一個分頁選「自訂網域」，依指示到你的網域商設定 DNS。

> ⚠️ 部署完成後請測試：直接在瀏覽器打開 `https://你的網址/taiwan` 這種子頁面。如果出現 404，回來告訴 Claude，加一個路由設定就能解決。

之後只要 GitHub 上的程式碼有更新（包含每月自動資料更新），Zeabur 都會**自動重新部署**。

### 替代方案：Netlify / Vercel（免費）

如果不想用掉 Zeabur 資源，Netlify 免費方案也很適合：https://app.netlify.com/signup → 用 GitHub 登入 → Import project → 選 `power-atlas` → Deploy。專案已附好 `public/_redirects` 與 `vercel.json` 設定。

---

## 第三步：確認每月自動更新資料有開啟

1. 到你的 GitHub repository 頁面，點上方「**Actions**」分頁。
2. 如果看到黃色提示，按「**I understand my workflows, go ahead and enable them**」。
3. 完成！之後每月 5 日，GitHub 會自動：下載最新的 OWID／IRENA 資料 → 通過驗證 → 提交 → Zeabur 自動重新部署。
4. 想立刻手動更新一次：Actions → 左邊選「Update data」→ 右邊「Run workflow」。

> 🛡️ 資料驗證失敗時（例如上游改了格式），更新會中止、網站保持舊資料不會壞，你會在 Actions 看到紅色叉叉——到時候再處理即可（或回來找 Claude 幫忙）。

## 之後想改內容怎麼辦？

| 想做的事 | 方式 |
|---|---|
| 更新台灣用電大戶、企業用電數字 | 編輯 `src/data/taiwan-detail.json`、`src/data/companies.json`（記得同步更新來源與年份），存檔後 `git add -A && git commit -m "update data" && git push` |
| 立刻抓最新國際資料 | 在專案資料夾執行 `npm run data -- --force`，然後 commit + push |
| 在本地電腦先預覽 | `npm run dev`，瀏覽器開 http://localhost:5173 |

## 常見問題

**Q：部署後會多花錢嗎？**
靜態網站幾乎不耗資源；放在你現有的 Zeabur 伺服器上不需要額外費用。GitHub 免費。

**Q：網站壞了怎麼辦？**
Zeabur 服務頁面可以查看部署歷史並回滾（Redeploy 之前的版本）；本機也永遠保有完整程式碼備份。

**Q：想暫時下線？**
在 Zeabur 把該服務暫停（Suspend）即可，隨時可以再啟動。
